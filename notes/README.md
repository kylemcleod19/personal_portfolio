# Notes

Working notes for the portfolio site. Claude reads these when helping update content.

## Structure

- `projects/` — notes on individual projects (one file per project)
- `blog-ideas/` — rough ideas and outlines for blog posts

## Usage

Drop a markdown file in the relevant folder. No required format, but useful things to include:

**For projects:**
- what it is and what it does
- the hypothesis or problem it was solving
- what role AI played
- what worked, what broke, what surprised you
- links (repo, live URL, demo)

**For blog ideas:**
- the core idea or argument
- any evidence or examples you have
- rough outline if you have one
- related projects or context

Claude will reference these when you ask for help updating the site.

## The Thesis Test

A product leader with deep domain knowledge and the right AI workflow can compress months of requirements, architecture, and implementation into days. ERPForge tests the sharpest version of that. Can you codify the product process itself into reusable AI skills, so the next system builds faster than the last?

## The Problem

Traditional ERPs exist because code was expensive. Vendors built lowest-common-denominator software for a business segment, then tried to configure it per customer. Big customers got the most features. Small ones got the base package or cobbled together Excel and point solutions.

At MacroFab, we built manufacturing workflows in-house because MRPs were expensive and slow to implement. The cost structure forced that tradeoff. Build it yourself or settle for software that doesn't match your operations.

What if the cost structure changed?

## The Approach

I architected the system with ChatGPT. Talked through the design on a walk, generated a `CLAUDE.md` file and an architecture doc, then used Claude to build the full platform in a day. The interesting part isn't the ERP. It's the layer on top: Claude skills for design review, discovery question templates, documentation generation. The skills *are* the product. The ERP is proof the process works.

- **`/design-review`** structured 10-section review before writing any code
- **Discovery templates** 80 questions across 12 categories for BOM and product scoping
- **`/write-docs`** auto-generates API reference, migration guides, and changelogs

The BOM rebuild is the clearest example. I gave Claude context on typical BOM structures and manufacturing processes, gave it skills around BOM information, and had it interview me about product workflows and configurations. Then it rebuilt the data model. Four hours from kickoff to a restructured, production-ready BOM system. Work that would normally take a systems integrator weeks.

## What It Does

Eight integrated modules: Products & BOM, Quotes, Orders, Inventory, Purchasing, Manufacturing, Shipping, and Invoices. The full quote-to-order-to-ship workflow runs end to end.

The BOM system is the deepest module. Recursive multi-level BOMs with revision control, alternate components, CSV import with column auto-detection, and depth-first cost rollup through the full tree.

## What's Actually Hard

Cloud hosting was new territory for me. Took a full day to get Railway + Cloudflare configured with containers, DNS, and security.

The next challenge: turning manual skills into automated agent workflows. Right now, Claude follows the skills when I prompt it. The goal is a system that runs the full interview, build, and iterate loop without manual orchestration. That's the difference between "I can build an ERP fast" and "anyone with domain knowledge can."

## What This Proves (So Far)

A product leader with the right AI workflow can stand up a realistic manufacturing ERP in days. But the bigger bet isn't one ERP. It's whether the skills and templates that built this one can build the next one faster, and eventually build themselves.
